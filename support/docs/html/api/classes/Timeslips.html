$relationships

$relationships : array

Type

array — Defines the relationships between models. Models are related together by defining the relationship in an array in the called model. The one exception to this is the HABTM (Has And Belongs To Many) relationship, which must be defined in both related models. The relationships get defined using the following structure: A has B - PK in A, FK in B A belongsTo B - PK in B, FK in A A HABTM B - FK in link table protected $relationships = array( 'alias' => array( // Alias to use for this table (use model name if no alias) 'model' => 'modelClass', // Name of the model (or model on other side of link table) 'type' => 'relType', // Type of relationship - has, belongsTo, HABTM 'localKey' => 'KeyField', // Local model PK () 'remoteKey' => 'KeyField' // PK or FK field name in related model (or local model's FK in link table) 'linkTable' => 'linkTableName' // Name of link table to use ) );

query()

query(string  $query, array  $options = array()) : mixed

Sends a query to the database server.

Parameters

string $query

The query to send to the server.
array $options

An array of options. Valid options are: returnAsArray - Returns an array containing the result set data. Valid values are true and false (default). fetch-style - Determines whether the array returned when using returnAsArray is numeric, associative (default), or both. Valid values are 'num', 'assoc', and 'both'.

Returns

mixed —

$result The default return value is a reference to the result set on the server.

On a query error or if the result set is empty, an associative array will be returned with two keys: status, which is an HTTP response code (400 if there is a query error or 404 if an empty result set), and message, which contains a text string with information about the status.

If the 'returnAsArray' option is set to true, the results will be returned as an array formatted per the fetch-style option.

Retrieve either a resource collection or an individual resource if a key is given.
